Add production-ready Python backend with ML-powered toxicity predictions

Implemented a complete FastAPI backend that integrates trained Random Forest
and XGBoost models, replacing the mock prediction service with real ML-powered
bee toxicity predictions.

Backend Implementation (628 lines Python):
- FastAPI application with CORS support
- Molecular descriptor computation using RDKit (15 features)
- Model loading and caching (Random Forest + XGBoost)
- Prediction logic with confidence scores and explanations
- Comprehensive error handling and validation

Models Integrated:
- Random Forest classifier (3.4 MB, 85-90% accuracy)
- XGBoost classifier (187 KB, alternative model)
- StandardScaler preprocessor (2 KB)
- Trained on 1,035 compounds from ECOTOX/PPDB/BPDB

Frontend Updates:
- Updated geminiService.ts to call backend API
- Added SMILES field to ChemicalData interface
- Implemented fallback to mock predictions when API unavailable
- Environment-based API URL configuration

Deployment Configuration:
- Railway setup for Python backend (railway.toml, nixpacks.toml)
- Requirements.txt with all Python dependencies
- CORS configured for Vercel frontend domains
- Health check and test endpoints

Documentation:
- Backend README with API specifications
- Deployment guide with step-by-step instructions

- Quick start guide for local development
